# 学习总结规则

## 全局声明

- `knowledge-digest-zh` 是放置在 `~/.agents/skills/learning/knowledge-digest-zh/` 的用户级全局 skill
- 它对所有工作区可用，不需要在项目内重复注册
- 当任务满足下述触发条件时，默认在最终回复阶段调用

## 触发条件

每当完成以下任意类型的任务后，**必须按照 `knowledge-digest-zh` 技能的模式A，在最终回复末尾附带「学习摘要」**：

- 完成一个项目/功能开发
- 做完一个实验或实验的某个阶段
- 回答了一个技术/学术问题
- 完成论文写作/翻译/分析
- 完成考试题目解答
- 任何涉及具体知识运用的任务

**不触发的情况**：纯文件操作（重命名/移动）、简单确认（"好的"）、skill 本身的维护修改、闲聊。

## 执行方式

1. 将 `knowledge-digest-zh` 视为全局默认学习总结技能并读取其 SKILL.md
2. 按模式A（自动摘要）的格式输出学习摘要
3. 用户主动要求深度总结时，切换到模式B
4. 用户说"考考我"时，切换到模式C

## skill-output 命名规则

- 新产物默认写入 `skill-outputs/<skill-name>/`，目录名已经表达调用的 skill。
- 文件或文件夹名使用中文主题 + 日期时间后缀，不再重复 skill 名。
- 单文件示例：`阶段审计报告_YYYYMMDD_HHMMSS.md`。
- 多文件交付示例：`学习总结_YYYYMMDD_HHMMSS/`。
- 主题不明确时使用 `结果_YYYYMMDD_HHMMSS` 或该 skill 明确规定的中文兜底名。
