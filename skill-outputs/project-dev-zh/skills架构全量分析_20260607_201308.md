## 1. 总体结论

当前 `skills/` 不是一个平铺提示词目录，而是一个“路由层 + 执行层 + 资源层 + 注册表生成层”的本地技能库。

核心特点：

- 本地共有 `44` 个 `SKILL.md`。
- 采用两层目录：`skills/<类型>/<skill-name>/SKILL.md`。
- `agent-skills-ao/` 提供通用软件工程生命周期技能，共 `22` 个。
- 中文本地场景技能共 `22` 个，覆盖科研、工程、课程、写作、调研、汇报、运维和守门规则。
- `skills/README.md` 是生成出来的注册表，不应手改。
- `skills/scripts/Update-SkillsReadme.ps1` 是 README 生成器。
- `skills/SKILL_ROUTING.zh-CN.md` 和 `guardrail/skill-router-zh/SKILL.md` 是路由入口。
- `skills/.instructions.md` 是维护规则，约束 README 同步、输出命名和编码。

当前主要维护债务：

1. `skills/README.md` 的行数统计已经落后于两个源技能。
2. 两个 `agents/openai.yaml` 不是 UTF-8 编码。

4. `ops/artifact-curator-zh/scripts/__pycache__/` 下存在 `.pyc` 缓存文件。

### 2.2 路由层

路由层负责“先决定用哪些技能，再加载最少必要技能”：

- `guardrail/skill-router-zh/SKILL.md`：中文总控路由技能。


### 2.4 资源层

资源层不是技能入口，而是被技能按需加载：

- `assets/`：模板、示例交付物、输入模板。
- `references/`：规则、检查表、经验沉淀。
- `agents/openai.yaml`：给特定技能提供 UI/agent 展示元数据。
- `scripts/`：技能内脚本或仓库级 README 生成器。

仓库级共享资料位于：

```text

  accessibility-checklist.md
  orchestration-patterns.md
  performance-checklist.md
  security-checklist.md

```

### 2.5 注册表生成层

`skills/README.md` 由 `skills/scripts/Update-SkillsReadme.ps1` 生成。生成器做了这些事：

- 扫描所有 `SKILL.md`。
- 读取 YAML frontmatter 的 `name` 和 `description`。
- 统计每个技能行数。
- 按 `scenarioBySkill` 分场景。
- 给 `agent-skills-ao` 技能补中文描述和触发词映射。
- 写出 Overview、Trigger Keywords、Common Features、Directory Structure。

维护规则要求：修改技能 frontmatter、描述、目录结构、行数显著变化或路由关系后，都应重新生成 README。

## 6. 发现的问题

### 6.1 README 注册表落后于源技能

`skills/README.md` 显示更新时间是 2026-05-22，但源文件有 2026-06-07 的改动。

行数漂移：

| Skill | README 行数 | 实际行数 | 差值 |
|---|---:|---:|---:|
| `iteration-reflection-guard-zh` | 152 | 173 | +21 |
| `performance-optimization` | 350 | 364 | +14 |

影响：

- README 的行数统计不准确。
- 如果 README 是外部索引或检索入口，可能会误导维护者判断技能规模和更新状态。
- 这不是 git 未提交 diff；当前 `git status --short` 只显示一个未跟踪 zip。

建议：

```powershell
cd C:\Users\zbh\.agents\skills
.\scripts\Update-SkillsReadme.ps1
```

### 6.2 两个 agent YAML 编码不符合 UTF-8 规则

按严格 UTF-8 校验失败的文本文件：

```text
skills/guardrail/iteration-reflection-guard-zh/agents/openai.yaml
skills/ops/artifact-curator-zh/agents/openai.yaml
```

这两个文件用 PowerShell 默认编码能正常读，但用 UTF-8 会出现替换字符。仓库规则写明“中文文件统一保存为 UTF-8”，所以它们需要转换为 UTF-8。

影响：

- 在严格 UTF-8 工具链或跨平台环境中可能显示乱码。
- 当前 agent 展示元数据可能在某些索引器中不可读。

### 6.3 根目录 zip 包过大且混杂

`skills/` 根目录下存在 5 个 zip：

```text
agent-skills-main.zip
basic-memory-main.zip
cc-sdd-main.zip
GPT-Image2-Skill-main.zip
superpowers-main.zip
```

其中 `GPT-Image2-Skill-main.zip` 约 423 MB，并且是当前 git 未跟踪文件。

影响：

- 会干扰 `skills/` 作为技能源码目录的清晰度。
- 大文件容易拖慢搜索、备份和索引。
- 未跟踪大文件如果误提交，会污染仓库。

建议：

- 将 zip 源包移到专门的归档目录，或在确认不需要后删除。
- 对 `*.zip` 建立明确 `.gitignore` 规则。
- 保留“技能源文件”和“下载缓存/安装包”的边界。

### 6.4 Python 缓存文件留在技能脚本目录

发现：

```text
skills/ops/artifact-curator-zh/scripts/__pycache__/auto_migrate_skill_outputs.cpython-314.pyc
skills/ops/artifact-curator-zh/scripts/__pycache__/curate_artifacts.cpython-314.pyc
```

影响：

- 对功能影响不大，但源码目录会混入运行缓存。
- 如果没有 `.gitignore` 保护，后续容易误提交。

建议：

- 清理 `__pycache__/`。
- 确认 `.gitignore` 覆盖 `__pycache__/` 和 `*.pyc`。

## 8. 当前架构的薄弱点

1. **索引依赖手动生成**  
   README 漂移说明“改完技能后同步 README”的规则没有被自动校验。

2. **编码一致性还没有自动门禁**  
   两个 YAML 已经偏离 UTF-8 规则，但没有被及时发现。

3. **资源归档边界不够硬**  
   zip、pyc、源码、技能入口混在 `skills/` 下，长期会影响搜索和维护。

4. **部分 `assets/` / `references/` 目录为空**  
   这不一定是问题，但会让目录树看起来比实际资源更复杂。

5. **路由入口存在两套语言体系**  
   `skill-router-zh` 和 `using-agent-skills` 都是元技能。当前分工是中文全库路由与英文工程生命周期路由，但维护时要避免二者规则冲突。
