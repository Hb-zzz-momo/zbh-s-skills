# Acceptance Checklist

联网时优先读取 Vercel 最新指南；本文件作为本地降级验收清单。

## Visual Quality

- [ ] 首屏清楚表达页面对象、目标用户和主要动作。
- [ ] 页面有明确视觉方向，不像通用模板。
- [ ] 配色不是单一色系堆叠，强调色全页一致。
- [ ] 字体层级清楚，中文页面行高和中文字体合理。
- [ ] 没有过度圆角、过度阴影、过度毛玻璃、过度渐变。
- [ ] 卡片、分隔、留白都服务信息结构。
- [ ] 页面至少有一处与主题相关的 signature moment。

## UX And Content

- [ ] CTA 文案具体，动作前后一致。
- [ ] 空状态、错误状态、加载状态完整。
- [ ] 表单有标签、校验、错误提示和成功反馈。
- [ ] 页面没有死路，必要时有返回、导航或下一步。
- [ ] 文案不使用 lorem ipsum、空泛营销语或占位废话。

## Accessibility

- [ ] 所有交互元素可键盘访问。
- [ ] 可见焦点态清楚。
- [ ] 图标按钮有 `aria-label` 或可见文本。
- [ ] 图片有有意义的 `alt`，装饰图为空 alt。
- [ ] 正文对比度达到 WCAG AA。
- [ ] 不只靠颜色表达状态。
- [ ] `prefers-reduced-motion` 有降级策略。

## Responsive And Layout

- [ ] 320px、768px、1024px、1440px 视口可用。
- [ ] 移动端无横向滚动。
- [ ] 长词、中文长句、按钮文案不溢出。
- [ ] 固定头部、浮层、弹窗不遮挡主要内容。
- [ ] 触摸目标至少 44px。

## Engineering

- [ ] 沿用现有框架、路由、样式方案和组件库。
- [ ] 新依赖已经在 `package.json` 中确认或显式安装。
- [ ] 组件职责清楚，数据获取和展示分离。
- [ ] 没有无用注释、死代码、调试日志。
- [ ] lint / typecheck / test / build 中可用项已执行。
- [ ] 浏览器控制台没有错误。

## Performance

- [ ] 大图有合理尺寸、懒加载或压缩策略。
- [ ] 动效只用 transform/opacity 等低成本属性为主。
- [ ] `pointermove`、scroll listener 做节流或 rAF。
- [ ] WebGL/重背景不超过一处，离屏暂停或移动端降级。
- [ ] 没有大面积移动元素使用 `filter: blur()`。
