# 项目启动规则

## 团队分工流程

1. 找几个人先读题，明确选什么题、解决什么问题、达到什么效果。
2. 搜资料，找可以复用的现成代码、论文、产品或竞品。
3. 从实际需求出发写用户侧应用方案和可行技术栈。
4. 先确定技术选型、前端设计、接口文档、数据库设计。
5. 再开始前端、后端、数据库和模型服务开发。

## 项目链路

常见项目链路：

1. 数据清洗
2. 调用封装好的核心函数
3. 错误识别和异常处理
4. 结果展示和导出

## 三类核心封装

- 存储类：判断集合是否存在、创建集合、定义对象结构、增删改查、查询逻辑。
- 编码/模型类：数据清洗、结构清洗、核心 encoding/model/client 调用。
- 核心业务类：对接上游 API、封装不同需求的查询、下载和表格构建函数。
