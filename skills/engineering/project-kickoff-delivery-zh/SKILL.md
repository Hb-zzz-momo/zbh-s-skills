# 项目启动交付技能

## 目标

把项目从“想做什么”推进到“团队知道怎么分工、代码知道怎么启动、交付知道怎么验收”。

## 启动流程

1. 读题和定位：明确要解决什么问题、用户是谁、最终交付物是什么。
2. 信息调研：找现成方案、竞品、论文/开源代码和可复用组件。
3. 痛点和应用方案：从实际需求出发写用户侧方案，而不是先堆技术。
4. 技术选型：前端、后端、数据库、模型服务、部署方式。
5. 接口和数据设计：先定接口文档、核心表、字段、状态流。
6. 团队分工：把任务按模块切给人，明确输入、输出和验收。
7. 启动链路：写清前端、后端、模型服务、数据库的启动命令。
8. 交付验收：功能闭环、演示路径、截图清单、风险清单。

## 输出要求

- 项目一句话定位
- 目标用户和痛点
- 技术栈与理由
- 模块分工表
- 接口/数据库初稿
- 启动命令
- 里程碑和验收标准

## 约束

- 先闭环，再扩展。
- 先真实可跑，再美化。
- 先定接口，再分头开发。
- 不要让团队成员同时改同一层的同一文件。
- 不要把模型服务、后端、前端的启动命令混在一起。
