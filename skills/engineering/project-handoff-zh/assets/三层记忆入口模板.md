# 三层记忆入口模板

适用于长期比赛、科研实验、性能优化、多分支工程。目标是让 AI 先读少量稳定入口，再按需追溯原文。

## 第一层：根规则文件

文件：`AGENTS.md` / `CLAUDE.md` / 项目根规则文件。

应包含：

- 启动读取顺序。
- 分支、远程、工作区规则。
- 验证 gate 和提交规则。
- 禁止事项。
- 必须先查第二层的歧义词或高风险概念。
- 本地/远程分支、worktree、保留例外和归档规则。

不应包含：

- 每轮实验流水。
- 大段结果表。
- 未验证猜测。

## 第二层：四个总览

建议目录：`资料/项目入口/` 或 `docs/project-entry/`。

### 计划总览.md

- 当前主目标。
- 当前强基线或冻结版本。
- 下一步优先级。
- 当前不做什么。
- 本轮 gate 和止损条件。

### 尝试方法总览.md

- 候选路线分类。
- 每类路线状态：可做、只做预审、禁止。
- 典型代表轮次或文档。
- 指向第三层原文。

### 结果总览.md

- 正式/官方结果表。
- 当前最优、稳定中位、负例。
- 数据解释规则。
- 不把单次高值当结论。

### 经验总览.md

- 已成立经验。
- 禁止重试路线。
- 歧义词表。
- 下次必须继承的判断规则。

## 第三层：详细档案

建议目录：

```text
计划/
尝试方法/
脚本/
结果/
经验/
```

每轮一个文件，文件名建议：

```text
YYYYMMDD_G编号_主题.md
YYYYMMDD_G编号_主题.json
```

第三层保存完整证据：命令、raw 路径、数据、失败原因、自审、下一步。

## 维护规则

- 第二层只写当前有效事实和跳转指针。
- 第三层保存完整历史，不要求每次全读。
- 新结论若影响启动判断，必须同步第二层。
- 复盘时必须区分：直接采纳、路线资产、负例禁止。
- 保持上下文干净：远程、本地分支、worktree 和 IDE 可见目录都应只保留冻结基线、当前最优和用户点名例外。
- 删除实验分支或 worktree 前，先保存 refs 清单和必要的 `git bundle --all` 归档。
