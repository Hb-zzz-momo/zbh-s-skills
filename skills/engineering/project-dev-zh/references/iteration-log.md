## 2026-04-23 迭代 #1

- **触发反馈**：用户希望把“逐段语法解析 + 业务作用说明 + 展开写法”这种代码教学风格吸收到 `project-dev-zh`。
- **根因**：原 `模式A：代码理解` 更偏逻辑与设计层讲解，缺少面向初学者的显式语法拆解规则，导致对压缩写法和逐行代码导读支持不足。
- **修改内容**：在 `SKILL.md` 的 frontmatter 和模式判定中补充“逐行讲解/语法解释”触发词；在模式A新增“代码教学增强规则”和“代码教学输出模板”。
- **影响范围**：模式A / 代码理解 / 初学者导向讲解

## 2026-04-23 迭代 #2

- **触发反馈**：用户要求今后 `project-dev-zh` 在知识点或语法分析时，都补简单例子帮助理解。
- **根因**：仅有定义、作用和展开写法还不够直观，初学者容易“看懂文字但不会迁移”，缺少最小例子会降低吸收速度。
- **修改内容**：在模式A的“代码教学增强规则”中新增“知识点/术语/语法点都要补简单例子”的硬规则；在代码教学输出模板里为“名词速览”和“语法点”新增“简单例子”字段。
- **影响范围**：模式A / 代码理解 / 语法教学输出

## 2026-04-23 迭代 #3

- **触发反馈**：用户指出“名词速览”不应停留在宽泛定义，而应说明术语在当前函数、当前调用点或当前会话里的具体意义。
- **根因**：此前模板要求了术语解释和简单例子，但没有强制把术语绑定到本地代码语境，导致输出可能退化为通用词典释义。
- **修改内容**：在模式A的“代码教学增强规则”中新增“名词速览必须解释当前代码语境中的具体作用”的硬规则；在输出模板里把“名词速览”改为“通用含义 + 在这里的作用 + 简单例子”。
- **影响范围**：模式A / 名词速览 / 代码教学上下文解释
