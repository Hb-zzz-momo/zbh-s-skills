# Skills 维护与路由规则

## 总控原则

本仓库不是平铺技能清单，而是按“先路由、再加载、少量组合”使用。

处理任何 skills 相关任务时：

1. 先读 `skills/README.md` 的场景分组和触发词。
2. 如果是“该用哪个技能 / 统筹 skills / 多技能组合”问题，优先使用 `skill-router-zh`。
3. 工程类任务参考 `agent-skills-ao/USAGE.zh-CN.md` 的 Define -> Plan -> Build -> Verify -> Review -> Ship 生命周期。
4. 一次只选择必要的 1-3 个 skill，复杂任务分阶段追加。
5. 有参考计划、题面、prompt 文件或用户明确边界时，优先加入 `plan-faithful-execution-zh`。

## README 自动同步

当对本仓库中任意技能目录（含类型子目录下的 `*/SKILL.md`）执行以下操作后，必须同步更新 `README.md`：

1. 新增 skill（在对应类型目录下新建技能文件夹 + `SKILL.md`）。
2. 删除 skill。
3. 修改任何 `SKILL.md` 的 `name`、`description`，或行数发生显著变化。
4. 技能关系、场景分组、协作关系发生变化。
5. 修改 `scripts/Update-SkillsReadme.ps1` 中的路由、场景或中文描述映射。

### 更新命令

```powershell
cd C:\Users\zbh\.agents\skills
.\scripts\Update-SkillsReadme.ps1
```

### 更新范围

- **技能总览表**：同步名称、行数、说明。
- **触发词速查**：同步 `Use when` 关键词。
- **路由协议**：保留 `SKILL_ROUTING.zh-CN.md` 和 `skill-router-zh` 入口。
- **目录结构**：新增或删除目录时同步树形图。
- **更新日期**：改为当天日期。

### 不需要更新 README 的情况

- 只修改 `assets/` 或 `references/` 中的普通材料。
- 只修正文档错别字且不影响 `SKILL.md` frontmatter、行数和目录结构。

## 编码规则

- 中文文件统一保存为 UTF-8。
- PowerShell 脚本如果包含中文哈希表或中文字符串，保持 UTF-8 BOM，避免 Windows PowerShell 解析异常。
- 发现 mojibake 时，优先修复源文件，再重新生成 `README.md`。
