## Global Defaults

- ``knowledge-digest-zh`` is a user-level global skill and is auto-invoked at task completion via ``~/.agents/.instructions.md``

## Skill Routing Protocol

- For any "which skill should I use" or multi-skill orchestration task, start with ``skill-router-zh``.
- For engineering tasks, use the ``agent-skills-ao`` lifecycle: Define -> Plan -> Build -> Verify -> Review -> Ship.
- For strict plans, contest prompts, fixed requirements, or user-specified boundaries, add ``plan-faithful-execution-zh`` before implementation.
- Load the minimum useful set: usually one primary skill plus one verification/review skill; avoid loading the whole library.
- Full routing handbook: ``SKILL_ROUTING.zh-CN.md``.

## Common Features

All skills support:

1. **Self-iteration mode** - give feedback after use to trigger optimization
2. **YAML frontmatter** - standard name / description fields
3. **Markdown deliverable** - substantial skill outputs must also be written to Markdown file(s) or a dedicated output folder in the current workspace
4. **Skill-output naming** - new outputs go under ``skill-outputs/<skill-name>/`` and use Chinese topic names with the date/time suffix, for example ``阶段审计报告_YYYYMMDD_HHMMSS.md``; do not repeat the skill name in the file or folder name
5. **Line limit** - every SKILL.md < 500 lines
