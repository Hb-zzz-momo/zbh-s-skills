# 考试答题辅助技能

## 技能目标

通过"3轮验证 + 置信度标注"机制，为考试/作业题目提供高可靠性解答，同时帮助用户真正理解而非死记硬背。

## Markdown 交付要求

1. 使用本技能完成正式输出时，除聊天中的简要说明外，必须在当前工作区新建一个 Markdown 文档保存完整结果。
2. 默认保存位置为当前工作区 `skill-outputs/exam-answer-zh/`（与 `artifact-curator-zh` 按技能分子目录一致；根目录为 **legacy-flat** 兼容）。若该子目录不存在须先创建。
3. 默认文件名格式为 `YYYYMMDD_HHMMSS_exam-answer-zh_主题.md`（置于上述目录下）；若主题不明确，使用 `result`。
4. Markdown 文档必须包含完整题目、解答过程、最终答案、复查结果与关键提示。
5. 若用户明确指定保存路径或文件名，以用户要求为准；若用户明确要求只在对话中回答，可跳过创建文件。
6. 最终回复中要说明新建的 Markdown 文件路径（须含 `skill-outputs/exam-answer-zh/` 前缀），以及文件里包含的主要内容。

## 3轮验证工作流

### 第一轮：解题

1. **审题**：识别题型（选择/填空/简答/计算/分析/编程）+ 关键条件 + 考察知识点
2. **解题**：按该题型的标准方法给出完整解答过程
3. **输出初步答案**

### 第二轮：教师视角复查

切换到"出题老师"的角度重新审视：

1. 这道题的标准答案(得分点)是什么？
2. 学生最容易犯的错误是什么？
3. 我的第一轮答案是否踩了这些坑？
4. 是否遗漏了关键得分点？
5. 若发现问题，修正答案

### 第三轮：置信度标注 + 易混点

对最终答案进行标注：

| 置信度 | 含义 | 标注 |
|--------|------|------|
| ★★★ | 确定正确 | ✅ 高置信 |
| ★★☆ | 大概率正确但有细节不确定 | ⚠️ 中置信（标明不确定的点） |
| ★☆☆ | 依赖记忆/可能有误 | ❓ 低置信（建议核实） |

同时输出：
- **易混点**：这道题常见的混淆概念
- **记忆锚点**：帮助记住关键结论的一句话提示

## 输出格式

```markdown
## 题目 [序号]

**题型**：[选择/简答/计算/...]
**考察知识点**：[...]

### 解答过程
[完整解题步骤]

### 最终答案
[答案] [置信度标注]

### 教师视角复查
- 标准得分点：...
- 常见错误：...
- 复查结论：[与初步答案一致 / 已修正]

### 易混点 & 记忆锚点
- 易混：...
- 记忆锚点：...
```

## 批量答题规则

当用户一次提交多道题时：

1. 按顺序逐题完成3轮验证
2. 每题都标注置信度
3. 最后附汇总表：

```markdown
| 题号 | 答案 | 置信度 | 备注 |
|------|------|--------|------|
```

## 特殊题型处理

### 选择题

- 先排除法缩小范围，再正向验证
- 对每个选项说明为什么对/错
- 多选题标注"是否可能漏选"

### 计算题

- 列出完整计算步骤，不跳步
- 关键中间结果单独标注
- 单位必须写

### 编程题

- 先写思路，再写代码
- 代码必须可运行
- 标注时间/空间复杂度

### 简答/论述题

- 按得分点分条作答
- 先核心观点，再展开论述
- 不写废话/套话

## 约束

1. **不编造知识点**——不确定时明确标注低置信
2. **不省略步骤**——解题过程完整，让用户能跟着理解
3. **中文作答**（除非题目要求英文）
4. 题目信息不足时先问清楚，不猜题意

## 自我迭代模式

### 触发条件

当用户在使用本技能后给出反馈（如"这部分不好用""输出格式需要改""多了/少了XX"），或主动说"优化这个skill"时，进入自我迭代模式。

### 迭代工作流

1. **收集反馈**：明确用户不满意的具体环节（哪个模式 / 哪个步骤 / 什么问题）
2. **诊断根因**：是规则缺失、规则冲突、粒度不够、还是场景未覆盖？
3. **提出修改方案**：列出拟修改的条目（原文 → 修改后），说明修改理由
4. **用户确认**：修改方案经用户确认后才执行
5. **写入 SKILL.md**：将修改直接应用到本技能文件
6. **记录迭代日志**：在 `references/iteration-log.md` 追加本次迭代记录

### 迭代日志格式

```markdown
## [日期] 迭代 #N

- **触发反馈**：...
- **根因**：...
- **修改内容**：...
- **影响范围**：模式X / 步骤Y
```

### 迭代约束

- 不破坏现有模式结构——增量修改，不推倒重写
- 每次迭代只改一个关注点——不趁机大改
- 修改后 SKILL.md 总行数仍 < 500
- 重大结构变更（如新增/删除模式）必须用户明确同意
