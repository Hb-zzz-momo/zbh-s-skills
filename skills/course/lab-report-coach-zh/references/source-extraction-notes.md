# Source Extraction Notes

## 这个 skill 是怎么提炼出来的

本技能从提示词库中 30+ 个实验相关提示词里抽取稳定骨架，覆盖课程包括：
数据可视化、大数据技术基础、操作系统、统计学、Python 程序设计等。

## v3 版本新增提炼来源

v3 在 v2 基础上，额外从以下提示词类型中提炼新增约束：

### 1. 手把手带练类（多课程实例）
- 统计学带练通用模板、操作系统实验三手把手、数据可视化各实验带练
- 共同特征：8 项固定步骤结构（目标/原理/文件/代码/解释/结果/截图/报错）
- 提炼结果 → SKILL.md 模式 A 的"逐步带练"8 项强制要求

### 2. 实验报告生成类
- python 生成实验报告、数据可视化生成实验报告、通用约束.md
- 共同特征：模板绝对忠诚（栏目名不改、编号不改、即使"输入样列"也保持原样）
- 共同特征：带练→报告的语气转写（教程口吻→实验报告口吻）
- 共同特征：代码不堆贴、不编造截图、占位格式统一
- 提炼结果 → SKILL.md 模式 B 的全部约束

### 3. Word 文档生成类
- word 报告限制.md、数据可视化"生成实验报告"提示词
- 共同特征：样式维度逐项列举（字体/字号/段前段后/行距/缩进/页边距等）
- 共同特征：在模板副本上填充，不新建空白 Word
- 提炼结果 → SKILL.md 模式 C

### 4. 实验复盘类
- 复盘学习.md
- 共同特征：不重做实验，而是深度回顾（本质解释/阶段划分/薄弱点/独立重做）
- 提炼结果 → SKILL.md 模式 D

### 5. 材料优先级
- 大数据技术基础"生成实验报告"、数据可视化"生成实验报告"
- 共同特征：始终存在明确的 5 级优先级（模板 > 带练 > 真实材料 > 课程要求 > 默认方案）
- 提炼结果 → SKILL.md "材料优先级"章节

### 6. 排错流程
- 操作系统实验三、统计学带练
- 共同特征：现象→原因→定位→修复→验证→写入报告
- 提炼结果 → SKILL.md "排错约束"章节新增第 6 项"报告写法"

## 没有保留的内容

以下内容没有固化进 skill，留给具体任务时动态填写：

- 某一次实验的具体题目（如"实验二 数据预处理""实验三 进程通信"）
- 某门课的专属章节编号、系统调用清单、R 函数清单
- 某次会话里的特定报错
- 某个同学版本的样例文本
- 某次作业的个人信息字段
- 某门课的代码语言（Python / C / R 等）

## 设计原则

1. 固定实验推进方法，降低长任务中途跑偏概率
2. 让输出同时覆盖操作、解释、截图、报告四条线
3. 模式分离：带练/报告/Word/复盘各有独立约束，不混用
4. 材料优先级明确：模板 > 带练 > 真实材料 > 课程要求 > 默认方案
5. 不替用户编造成果，不确定的内容显式标注
# Source Extraction Notes

## 这个 skill 是怎么提炼出来的

这个技能不是把某一条长提示词原样复制进来，而是从一类高频任务里抽出了稳定骨架。

原始提示词的共同特征主要有：

1. 用户经常要求“手把手带我完成实验”
2. 用户不仅要操作步骤，还要知道为什么这么做
3. 用户经常同时需要：
   - 环境排错
   - 命令解释
   - 截图安排
   - 实验报告正文
4. 用户不喜欢只回答局部，希望必要时给出“修订版完整流程”
5. 用户强调不要编造成果，要区分已验证和未验证内容

## 从原提示词抽出的稳定层

提炼时主要保留了这几层：

- 触发场景
- 禁用场景
- 输入材料清单
- 固定执行流程
- 报错处理方法
- 实验报告输出约束
- 长任务进度汇报

## 没有保留的内容

以下内容没有固化进 skill，而是留给具体任务时动态填写：

- 某一次实验的具体题目
- 某门课的专属章节编号
- 某次会话里的特定报错
- 某个同学版本的样例文本
- 某次作业的个人信息字段

## 设计原则

这个 skill 的设计目标不是“替用户直接写完所有内容”，而是：

1. 固定实验推进方法
2. 降低长任务中途跑偏概率
3. 让输出兼顾操作、解释、截图、报告四条线
4. 让结果能直接进入 Codex / ChatGPT 的长期复用场景
