# 指标红旗清单

## 通用红旗

- 指标名清楚但实现没读。
- 指标方向没确认：越大越好和越小越好混用。
- 指标输入对象没确认：最终种群、非支配解、archive、best、summary 行可能不同。
- collector 做了过滤、去重、补缺失、均值化，但报告没说明。
- summary 数字无法从 raw 文件重算。
- 不同 step、driver、时间批次、结果根目录混用后直接排名。
- 缺失结果被静默忽略。
- run 列表不一致却比较均值。
- seed 来源不明，或 `run` 没有真正控制随机种子。

## IGD 红旗

- 未知 PF 问题却直接使用默认 IGD。
- `GetOptimum()` 未覆盖，框架返回默认点或空参考集。
- 参考点是坏点，例如最小化问题里使用 `[1,1]` 作为单点参考。
- 不同算法使用了不同 PF/reference set。
- PF* 来自离线合并时，未说明包含哪些算法和 runs。
- 把默认 IGD 和 PF* IGD 混在同一张主表里。

## HV 红旗

- reference point 未说明。
- 目标方向未确认，最小化问题未做正确归一化或参考点选择。
- 不同数据集的 HV 直接平均但尺度或参考点不一致。
- HV 高但最终非支配解数极低，需要检查是否只靠单端点撑起体积。

## 排名和胜负红旗

- 平均排名、逐数据集胜负、pair win 使用不同指标口径。
- 只报均值，不报每个数据集的 win/loss。
- 先看结果后改 gate。
- 旧表中的 stale 指标没有降级标注。

## 论文表红旗

- 主表指标没有可复算脚本。
- 结果文件路径不可追溯。
- 正式 claim 依赖历史混合结果。
- 指标修正后没有同步更新 workflow、tracing 或说明文件。
