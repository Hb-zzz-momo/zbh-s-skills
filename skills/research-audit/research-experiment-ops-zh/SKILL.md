# 科研实验运营技能

## 目标

把科研算法实验从“想到就跑”改成可复现闭环：假设定义 → smoke 验证 → dev 小批量 → formal 正式跑 → 汇总解析 → 消融/审计 → 下一轮决策。

## 工作流

1. 明确本轮唯一主假设：本轮只回答一个机制问题，避免多机制混改。
2. 锁定口径：问题、数据集、对照算法、指标、`N/maxFE/run/save`、结果目录。
3. 先跑 smoke：小规模验证可运行、无 NaN、无维度错误、日志字段存在。
4. 再跑 dev：少量 run 验证趋势，不把 dev 集调成过拟合。
5. 通过 gate 后跑 formal：正式 run、固定参数、只收集不再改公式。
6. 汇总结果：同时看主指标、任务层指标、runtime 和失败队列。
7. 输出结论：通过、回滚、止损或进入下一轮；同步 tracing/报告。

## 默认 gate

- Smoke：能跑完、无整批 NaN、结果文件存在、关键日志字段可读。
- Dev：相对冻结版不系统性退化；至少一个目标指标改善；runtime 不超过预设上限。
- Formal：满足预先写下的胜出条件；不得事后改成功标准。

## 输出要求

每次正式使用本技能，输出应包含：

- 实验目标和单主假设
- 数据集/算法/指标/参数表
- smoke/dev/formal 的运行入口
- 通过/拒绝 gate
- 结果文件和汇总文件位置
- 下一步动作：继续、回滚、止损或转论文整理

## 约束

- 不要一轮同时改多个主机制。
- 不要只追均值提升，要看逐数据集逐指标。
- 不要用开发集连续调参后直接宣称泛化。
- 不要忽略 runtime 和失败率。
- 不要只报好看的 HV/IGD，不解释 MinError、subset ratio、unique subsets 等任务层指标。
